# ClawX 授权验证与分发安全方案

> 版本：v2.0（已实现）
> 日期：2026-05-13
> 适用范围：ClawX 桌面客户端（Electron + React + Vite）
> 目标：将开源项目转为闭源分发，实现基于机器码的离线授权验证

---

## 1. 整体安全架构

采用 **"多层防御"（Defense in Depth）** 策略，不依赖单一防护点：

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 1: 代码分发保护                                       │
│  • electron-builder asar 加密                                │
│  • javascript-obfuscator 重度混淆                            │
│  • 可执行文件代码签名 (可选)                                  │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: 反逆向与反调试                                     │
│  • 反 DevTools 检测                                          │
│  • 反 debugger 附加                                          │
│  • 运行时代码完整性校验 (哈希校验关键文件)                      │
├─────────────────────────────────────────────────────────────┤
│  Layer 3: 核心验证逻辑保护 (C++ N-API)                        │
│  • Ed25519 公钥 XOR 变形硬编码                               │
│  • 验证逻辑在原生二进制中执行                                  │
│  • 反调试检测（原生层）                                       │
├─────────────────────────────────────────────────────────────┤
│  Layer 4: 机器指纹绑定                                       │
│  • 多因子硬件信息采集 (CPU/主板/BIOS/磁盘/MAC)                 │
│  • 主指纹 SHA256 计算，取前 8 字节作为机器码                   │
│  • 授权码与机器码精确匹配（8 字节指纹）                        │
├─────────────────────────────────────────────────────────────┤
│  Layer 5: 授权数据本地保护                                    │
│  • electron-store AES-256-GCM 加密存储                       │
│  • license 与机器指纹二次绑定                                │
│  • 防复制：license 中的机器指纹需与当前设备匹配                 │
└─────────────────────────────────────────────────────────────┘
```

## 2. 机器指纹生成算法（Layer 4）

### 2.1 采集的硬件因子

采集 **6 个硬件因子**：

| 优先级 | 因子 | 来源 | 稳定性 | 获取方式 |
|--------|------|------|--------|----------|
| P0 (核心) | CPU 序列号 | `wmic cpu get ProcessorId` / `sysctl machdep.cpu` | 高 | C++ 系统调用 |
| P0 | 系统盘序列号 | `wmic diskdrive get SerialNumber` / `diskutil info` | 高 | C++ 系统调用 |
| P1 (重要) | 主板 UUID | `wmic csproduct get UUID` / `IOPlatformUUID` | 高 | C++ 系统调用 |
| P1 | BIOS/系统序列号 | `wmic bios get SerialNumber` / `SPHardwareDataType` | 中 | C++ 系统调用 |
| P2 (辅助) | 主网卡 MAC | `GetAdaptersInfo` / `sysctl NET_RT_IFLIST` | 中 | C++ 网络 API |
| P2 | 主机名 | `os.hostname()` | 低 | Node.js 内置 API |

### 2.2 指纹生成算法

```cpp
std::string MachineFingerprint::HashString(const std::string& input) {
  unsigned char hash[SHA256_DIGEST_LENGTH];
  SHA256((const unsigned char*)input.c_str(), input.length(), hash);

  std::ostringstream oss;
  for (int i = 0; i < SHA256_DIGEST_LENGTH; i++) {
    oss << std::hex << std::setw(2) << std::setfill('0') << (int)hash[i];
  }
  return oss.str();
}

std::vector<unsigned char> MachineFingerprint::ComputeMasterFingerprint() {
  std::vector<std::string> hashes;
  hashes.push_back(HashString(GetCpuId()));
  hashes.push_back(HashString(GetDiskId()));
  hashes.push_back(HashString(GetBoardUuid()));
  hashes.push_back(HashString(GetBiosSn()));
  hashes.push_back(HashString(GetMacAddress()));
  hashes.push_back(HashString(GetHostname()));

  std::sort(hashes.begin(), hashes.end());
  std::string combined;
  for (const auto& h : hashes) {
    combined += h;
  }

  unsigned char finalHash[SHA256_DIGEST_LENGTH];
  SHA256((const unsigned char*)combined.c_str(), combined.length(), finalHash);
  return std::vector<unsigned char>(finalHash, finalHash + SHA256_DIGEST_LENGTH);
}
```

1. 每个因子单独 SHA256 哈希，统一长度
2. 所有因子哈希按字典序拼接后再次 SHA256，得到 **主指纹**（32 字节）
3. 展示给用户的机器码：主指纹前 **8 字节** 大写十六进制，每 4 字符分组
   - 格式：`XXXX-XXXX-XXXX-XXXX`（16 字符，不含前缀）
   - 示例：`3110-4974-9A41-8937`

### 2.3 授权匹配策略：精确匹配

**重要变更（v2.0）**：从"6 因子加权容错匹配（85% 阈值）"简化为"8 字节机器指纹精确匹配"。

原因：紧凑的 20 字节二进制 payload 无法容纳 6 个 32 字节哈希值，且精确匹配更适合早期 beta 阶段。

```cpp
// 验证时直接比较 payload 中的 8 字节指纹与当前设备指纹
std::vector<unsigned char> currentFp = MachineFingerprint::ComputeMasterFingerprint();
bool fpMatch = (memcmp(lp.machine_fp, currentFp.data(), 8) == 0);
```

**场景覆盖：**

| 操作 | 结果 |
|------|------|
| 换网卡、改主机名 | **不通过**（指纹变化） |
| 重装系统（磁盘不变） | **不通过**（指纹变化） |
| 换主板 | **不通过**（需重新授权） |
| 换 CPU 或换硬盘 | **不通过**（需重新授权） |

> 注：正式版可考虑加入"授权转移"流程（基于旧 license + 邮件验证）

### 2.4 跨平台实现

| 抽象因子 | Windows 来源 | macOS 来源 |
|---------|-------------|-----------|
| `cpuId` | `__cpuid` / `wmic` | `sysctl machdep.cpu.brand_string` |
| `diskId` | `GetVolumeInformationA` / `wmic` | `diskutil info` Volume UUID |
| `boardUuid` | `wmic csproduct UUID` | `IOKit` `IOPlatformUUID` |

| `hostname` | `gethostname()` | `gethostname()` |

## 3. 授权码生成与验证算法（Layer 3 + Layer 5）

### 3.1 密钥体系

- **私钥**：Ed25519，开发者持有，用于签名生成 License，**绝不泄露、不提交到版本控制**
- **公钥**：Ed25519（32 字节原始密钥），经过 XOR 变形后硬编码在 C++ 原生模块中，用于客户端验证签名
- **安全性**：Ed25519 等效于 ~3072-bit RSA 安全级别，签名仅 64 字节

**密钥生成命令：**
```bash
openssl genpkey -algorithm ED25519 -out private.pem
openssl pkey -in private.pem -pubout -out public.pem
```

### 3.2 License 数据结构

**格式：**
```
<payload_base58>.<signature_base58>
```

- **Payload**（20 字节二进制）：Base58 编码后约 28 字符
- **Signature**（64 字节）：Base58 编码后约 88 字符
- **总长度**：约 **115 字符**（旧方案约 1000 字符）

**20 字节 Payload Schema：**

| 字节 | 字段 | 类型 | 说明 |
|------|------|------|------|
| 0 | `version` | uint8 | License 协议版本，固定 `0x01` |
| 1-2 | `product_id` | uint16 BE | 产品 ID，固定 `0x0001` |
| 3 | `edition` | uint8 | `0x01` standard, `0x02` pro, `0x03` enterprise |
| 4-7 | `expiry_days` | uint32 BE | 过期天数（从 2024-01-01 起算），`0` = 永久 |
| 8-15 | `machine_fp` | uint8[8] | 机器指纹前 8 字节 |
| 16-19 | `serial` | uint32 BE | 随机序列号（内部追踪） |

```cpp
#pragma pack(push, 1)
struct LicensePayload {
  uint8_t  version;      // 0x01
  uint16_t product_id;   // 0x0001
  uint8_t  edition;      // 0x01/0x02/0x03
  uint32_t expiry_days;  // days since 2024-01-01, 0 = perpetual
  uint8_t  machine_fp[8];
  uint32_t serial;
};
#pragma pack(pop)
static_assert(sizeof(LicensePayload) == 20, "LicensePayload must be 20 bytes");
```

### 3.3 签名验证流程（C++ N-API）

```cpp
Napi::Value LicenseVerifier::Verify(const Napi::CallbackInfo& info) {
  // 1. 反调试检测
  if (AntiDebug::IsDebuggerPresent()) {
    throw SecurityViolation();
  }

  // 2. 解析 License 字符串（按 '.' 分割，Base58 解码 payload 和 signature）
  std::string licenseString = info[0].As<Napi::String>().Utf8Value();
  std::vector<unsigned char> payloadBytes, signatureBytes;
  if (!ParseLicense(licenseString, payloadBytes, signatureBytes)) {
    return Napi::Boolean::New(env, false);
  }

  // 3. 解析二进制 Payload
  LicensePayload lp;
  memcpy(&lp, payloadBytes.data(), sizeof(lp));

  // 4. 检查版本和产品 ID
  if (lp.version != 0x01) return false;
  if (ntohs(lp.product_id) != 0x0001) return false;

  // 5. 检查过期时间
  uint32_t expiryDays = ntohl(lp.expiry_days);
  if (!CheckExpiry(expiryDays)) return false;

  // 6. 公钥去混淆并加载
  EVP_PKEY* pkey = CryptoUtils::LoadDeobfuscatedPublicKey();

  // 7. Ed25519 签名验证（NULL digest，Ed25519 内部使用 SHA-512）
  bool sigValid = CryptoUtils::Ed25519Verify(
    pkey, payloadBytes.data(), payloadBytes.size(),
    signatureBytes.data(), signatureBytes.size()
  );
  EVP_PKEY_free(pkey);

  if (!sigValid) return false;

  // 8. 机器指纹精确匹配（8 字节）
  std::vector<unsigned char> currentFp = MachineFingerprint::ComputeMasterFingerprint();
  return (memcmp(lp.machine_fp, currentFp.data(), 8) == 0);
}
```

**过期检查实现：**
```cpp
bool LicenseVerifier::CheckExpiry(uint32_t expiryDays) {
  if (expiryDays == 0) return true; // 永久授权

  struct tm base = {};
  base.tm_year = 2024 - 1900;
  base.tm_mon = 0;
  base.tm_mday = 1;
  base.tm_isdst = -1;

  time_t baseTime = mktime(&base);
  time_t now = time(NULL);
  uint32_t currentDays = (uint32_t)(difftime(now, baseTime) / (24.0 * 3600.0));

  return currentDays <= expiryDays;
}
```

### 3.4 公钥保护策略（核心！）

**C++ 原生层实现：**

Ed25519 公钥（32 字节原始密钥 + ASN.1 包装后约 113 字节 PEM）经过 XOR 编码后硬编码在 `crypto_utils.cc` 中：

```cpp
static const unsigned char kObfPubKey[] = {
  0x3B, 0x0A, 0x31, ... // 113 bytes, XOR'd with 0x5A
};

EVP_PKEY* CryptoUtils::LoadDeobfuscatedPublicKey() {
  std::vector<unsigned char> decoded;
  for (size_t i = 0; i < sizeof(kObfPubKey); i++) {
    decoded.push_back(kObfPubKey[i] ^ 0x5A);
  }

  BIO* bio = BIO_new_mem_buf(decoded.data(), decoded.size());
  EVP_PKEY* pkey = PEM_read_bio_PUBKEY(bio, NULL, NULL, NULL);

  OPENSSL_cleanse(decoded.data(), decoded.size());
  BIO_free(bio);
  return pkey;
}
```

**增强措施：**
- 公钥 XOR 编码后硬编码在 `.node` 二进制中
- 验证完成后立即 `OPENSSL_cleanse` 清理内存
- 公钥重组函数在编译后被机器码优化，无清晰的函数边界

### 3.5 本地授权数据存储加密（Layer 5）

```javascript

const crypto = require('crypto');

function deriveStorageKey(machineFingerprint) {
  const salt = 'ClawX-Fixed-Salt-v1';
  return crypto.pbkdf2Sync(machineFingerprint + salt, salt, 100000, 32, 'sha256');
}

const store = new Store({
  name: 'license',
  encryptionKey: deriveStorageKey(getMachineFingerprint().fingerprint),
});

function saveLicense(licenseString, payload) {
  store.set('license', licenseString);
  store.set('activated_at', Date.now());
  store.set('last_check', Date.now());
}
```

**防护要点：**
- 加密密钥由机器指纹 + 固定盐 PBKDF2 派生
- `license` 文件复制到其他机器无法解密
- 额外存储 `activated_at` 和 `last_check`，可检测系统时间回拨

### 4.4 Base58 编解码实现

```cpp
static const char BASE58_ALPHABET[] =
  "123456789ABCDEFGHJKLMNPQRSTUVWXYZabcdefghijkmnopqrstuvwxyz";

bool CryptoUtils::Base58Encode(const std::vector<unsigned char>& data, std::string& out) {
  if (data.empty()) { out = ""; return true; }

  int leadingZeros = 0;
  while (leadingZeros < (int)data.size() && data[leadingZeros] == 0) leadingZeros++;

  std::vector<int> digits;
  for (size_t i = 0; i < data.size(); i++) {
    int carry = data[i];
    for (size_t j = 0; j < digits.size(); j++) {
      carry += 256 * digits[j];
      digits[j] = carry % 58;
      carry /= 58;
    }
    while (carry > 0) {
      digits.push_back(carry % 58);
      carry /= 58;
    }
  }

  out.clear();
  for (int i = 0; i < leadingZeros; i++) out += BASE58_ALPHABET[0];
  for (int i = digits.size() - 1; i >= 0; i--) out += BASE58_ALPHABET[digits[i]];
  return true;
}
```

### 4.5 反调试（原生层）

```cpp
bool AntiDebug::IsDebuggerPresent() {
#if defined(_WIN32)
  if (::IsDebuggerPresent()) return true;

  BOOL remoteDebugger = FALSE;
  CheckRemoteDebuggerPresent(GetCurrentProcess(), &remoteDebugger);
  if (remoteDebugger) return true;

  auto start = GetTickCount64();
  volatile int dummy = 0;
  for (int i = 0; i < 1000000; i++) dummy++;
  return (GetTickCount64() - start) > 1000;

#elif defined(__APPLE__)
  struct kinfo_proc info;
  size_t info_size = sizeof(info);
  int mib[4] = { CTL_KERN, KERN_PROC, KERN_PROC_PID, getpid() };

  if (sysctl(mib, 4, &info, &info_size, NULL, 0) == 0) {
    if (info.kp_proc.p_flag & P_TRACED) return true;
  }

  auto start = clock();
  volatile int dummy = 0;
  for (int i = 0; i < 1000000; i++) dummy++;
  return (clock() - start) * 1000 / CLOCKS_PER_SEC > 1000;

  return false;
}
```

### 6.3 CLI 授权生成工具

```javascript
// scripts/generate-license.mjs
function packPayload(machineCode, days, edition) {
  const payload = Buffer.alloc(20);
  payload.writeUInt8(0x01, 0);                    // version
  payload.writeUInt16BE(0x0001, 1);               // product_id
  payload.writeUInt8({ standard: 0x01, pro: 0x02, enterprise: 0x03 }[edition] || 0x01, 3);

  let expiryDays = 0;
  if (days > 0) {
    const base = new Date('2024-01-01T00:00:00Z');
    const now = new Date();
    expiryDays = Math.floor((now - base) / (1000 * 60 * 60 * 24)) + days;
  }
  payload.writeUInt32BE(expiryDays, 4);           // expiry_days

  const fp = machineCodeToFingerprint(machineCode); // 8 bytes
  fp.copy(payload, 8, 0, 8);                      // machine_fp
  payload.writeUInt32BE(Math.floor(Math.random() * 0xFFFFFFFF), 16); // serial

  return payload;
}

function generateLicense(privateKeyPem, machineCode, days, edition) {
  const payload = packPayload(machineCode, days, edition);
  const privateKey = createPrivateKey(privateKeyPem);
  const signature = sign(null, payload, privateKey); // Ed25519, null digest
  return `${base58Encode(payload)}.${base58Encode(signature)}`;
}
```

## 10. 风险与应对

| 风险 | 可能性 | 影响 | 应对 |
|------|--------|------|------|
| C++ 跨平台编译问题 | 中 | 高 | 已解决：macOS arm64/x64 通用二进制构建通过 |
| N-API ABI 兼容性 | 中 | 中 | 锁定 Electron 版本，升级时同步 recompile |
| 混淆后性能下降 | 低 | 中 | 控制流扁平化阈值可调（0.7-1.0） |
| 用户硬件大改无法使用 | 低 | 低 | 当前精确匹配方案，需重新授权（后续可加转移流程） |
| 专业逆向团队破解 | 中 | 高 | C++ 原生层 + 加壳 + 持续更新验证算法 |
| 系统时间回拨绕过过期 | 中 | 中 | 已记录 `activated_at` 和 `last_check`，可检测异常 |

## 11. 安全等级评估

| 攻击方式 | 纯 JS 方案 | C++ N-API 方案（本方案） |
|---------|-----------|------------------------|
| 直接提取公钥 | 中等 | **高**（XOR 变形 + 二进制硬编码） |
| 内存 Dump 公钥 | 低 | **中等**（验证后清理内存） |
| Patch 跳过验证 | 极低 | **高**（需在二进制中 patch 汇编） |
| 制作注册机 | 高 | **极高**（不知道私钥几乎不可能） |
| 预期破解时间 | 2-8 小时 | **数天到数周** |

## 12. 版本变更记录

### v2.0（2026-05-13）- 已实现

**重大变更：**
- **签名算法**：RSA-2048 + SHA-256 → **Ed25519**（签名 64 字节，等效 ~3072-bit RSA）
- **Payload 格式**：JSON + base64url（~300 字节） → **20 字节二进制 + Base58**（~28 字符）
- **License 长度**：~1000 字符 → **~115 字符**
- **机器码格式**：`CLAWX-XXXX-XXXX-XXXX-XXXX` → **`XXXX-XXXX-XXXX-XXXX`**（移除前缀，未来可改名）
- **机器匹配**：6 因子加权容错（85% 阈值） → **8 字节指纹精确匹配**
- **用户输入**：需提供 6 个硬件因子 JSON → **仅需 16 字符机器码**
- **注册机**：新增独立 Electron GUI 应用 `tools/license-admin/`

**新增功能：**
- 授权过期检查（基于 2024-01-01 的天数计数）
- 独立 License Admin 工具（生成/历史/导出）
- 激活成功后自动启动 Gateway（修复了之前未启动的 bug）
